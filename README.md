## 1. 创意名称 + 创意介绍

* 想解决什么问题：中国有 3.23 亿 60 岁以上老年人，AI 换脸/拟声诈骗同比暴涨 380%，60 岁以上受害人占 67%。深伪技术让骗子能伪造亲人的脸和声音——老人不是变笨了，是判断真假的坐标系被摧毁了。Brown University 2025 年发表在 USENIX SOUPS 的研究揭示：老人判断诈骗的唯一方式，是依赖"可信关系"——这个人我认不认识、这个声音我熟不熟悉。而深伪技术恰恰攻击的，就是这个唯一的锚点。

* 为什么会想到做这个：身边长辈几乎都经历过冒充亲友、医保诈骗的套路。骗子会特意说"别告诉家人"，而老人往往因为"不好意思麻烦子女"独自承受。既然老人天然依赖家人来做判断，产品就不该把"家属确认"当成附属按钮，而应该把它做成整个产品的中心轴。

* 大概是什么产品：一个移动端优先的 Web App，核心逻辑是"AI 帮你整理，家人帮你确认"——大字、短句、零术语、一键转发给子女。

## 2. 目标用户及痛点

**面向哪些用户：**
* 60 岁以上有基本手机操作能力的老年人；
* 长期远程关心父母的子女；
* 社区/养老机构反诈服务者。

**在什么场景下使用：**
* 独居老人收到医保异常短信。
* 接到"孙子"视频哭着借钱、子女不在身边来不及商量。
* 用手机挂号/办医保不知从何下手。

**当前痛点：**
* 没有产品时，老人判断全凭直觉和临时电话求助。但深伪诈骗就是针对"来不及求助"的时间空档设计的。受骗后的代价不止是钱——三联生活周刊报道的案例中，外婆被骗 3 万后不敢再用座机，换了只存家人号码的老年机。被骗走的不止是养老金，是老人在数字社会里的安全感和对子女的信任。
## 3. 价值与意义
* 所有冒充亲友类诈骗有一个共同底层逻辑——隔离受害者与ta信任的人。骗子会说"保密""别告诉子女"。银发守护 AI 的根本设计是对这个逻辑的反制：每一次交互的终点，都是"转发给子女看看"。四项核心功能全部以连接家人为终点——识别风险 → 缓冲冲动 → 一键转发求助 → 办事陪练。

## 快速开始

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build
```

打开 http://localhost:3000

### 使用真实 AI API

```bash
# 设置环境变量（DeepSeek 为例）
export NEXT_PUBLIC_DEEPSEEK_API_KEY=your_api_key
export NEXT_PUBLIC_DEEPSEEK_BASE_URL=https://api.deepseek.com
export NEXT_PUBLIC_DEEPSEEK_MODEL=deepseek-chat
```

不设置 API Key 时，自动使用内置 Mock 数据——也能完整体验全部功能。
